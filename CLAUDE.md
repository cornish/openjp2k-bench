# openjp2k-bench — Claude context

## What this is

C++/CMake benchmark harness for JPEG 2000 decode performance. Compares:

- **OpenJPEG** (vanilla, from upstream) — the baseline.
- **Grok** — the only meaningful open-source comparison point.
- **openjp2k** (a fork of OpenJPEG) — built in via `--openjpeg-source`.

Kakadu has no adapter (no open binary available); its published
numbers can be pasted into reports manually.

## Build, run, report

```sh
scripts/setup.sh                              # clones openjpeg + grok at pinned refs
scripts/build.sh                              # baseline build with vanilla openjpeg
scripts/build.sh --openjpeg-source ../openjp2k # build against the fork
scripts/run_bench.sh <corpus-dir>             # dumps JSON results
scripts/report.py <baseline.json> <new.json>  # comparison table + diff
```

Build flags worth knowing:
- `--no-grok` — skip the Grok adapter.
- `--no-native` — drop `-march=native` (for portable / cross-machine runs).
- `--openjpeg-source <path>` — swap in a local fork instead of pinned upstream.

## Corpus

Not checked in. Generated by `scripts/gen_corpus.sh`, which drives
`opj_compress` to produce parametric inputs across sizes, lossy/
lossless, decomposition levels, and tile modes. Re-generate locally
on each benchmark machine to control for codec/version drift.

## Reporting conventions

Each result JSON should record:
- Decoder name + library version.
- Source-repo commit SHA (so a perf change can be traced to a fork commit).
- Hardware identifier (CPU model, thread count).
- Compile flags actually used.

Without those, reports comparing across machines/builds are noise.

## What the bench measures

Per the original methodology doc (`docs/methodology.md`): warmup + N
iterations, reporting min/p50/p90/p99/mean/stddev. Wall time of the
decode call only — not file I/O or output write. Cross-decoder pixel
verification confirms the decoders produce equivalent output before
trusting the timing numbers.

## Sibling repos

- **[cornish/openjp2k](https://github.com/cornish/openjp2k)** — the
  performance fork under test. Point `--openjpeg-source` at a local
  checkout to bench it.
- **[cornish/openjp2k-data](https://github.com/cornish/openjp2k-data)** —
  owns the curated public corpus (conformance / archival / remote-sensing /
  medical / cinema) plus `MANUAL_SOURCES.md` for auth-gated sources.
  `corpus/public/` in this repo is a symlink to that repo's `corpus/`. See
  `corpus/README.md` for setup.
- **[cornish/openscope](https://github.com/cornish/openscope)** — the
  WSI viewer whose tile-decode pattern (one tile, often partial region)
  this benchmark is shaped to mimic. Real workload validation happens
  there.
